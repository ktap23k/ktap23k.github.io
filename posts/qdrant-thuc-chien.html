Qdrant thực chiến — vector database mạnh mẽ và dễ dùng

TA Tuan Anh · 5 tháng 4, 2026 · ⏱ 11 phút đọc

Khi xây dựng ứng dụng AI hiện đại — đặc biệt là RAG và recommendation — bạn cần một nơi lưu trữ và tìm kiếm vector hiệu quả. Trong khi nhiều người nghĩ ngay đến Pinecone hoặc Weaviate, mình muốn giới thiệu Qdrant: một vector database mã nguồn mở, viết bằng Rust, chạy nhanh, dễ deploy và có Python client rất tiện.

Tại sao chọn Qdrant?

Qdrant không phải là lựa chọn duy nhất, nhưng nó có vài điểm mạnh khiến mình thích:

  • Mã nguồn mở: có thể self-host miễn phí, không bị lock-in.
  • Hiệu năng: viết bằng Rust, sử dụng HNSW để tìm kiếm gần đúng nhanh.
  • Payload filtering: kết hợp vector search với metadata filter mạnh mẽ.
  • Hybrid search: hỗ trợ kết hợp dense vector và sparse vector (BM25).
  • Dễ deploy: Docker image nhẹ, hoặc dùng Qdrant Cloud.

Cài đặt và chạy Qdrant

Cách nhanh nhất để thử Qdrant là Docker:

docker run -p 6333:6333 -p 6334:6334 \
  -v $(pwd)/qdrant_storage:/qdrant/storage \
  qdrant/qdrant

Port 6333 là REST API, 6334 là gRPC. Nếu dùng Python, gRPC thường nhanh hơn đáng kể cho batch operations.

Các khái niệm cốt lõi

Collection tương đương table trong relational database. Mỗi collection có một vector configuration cố định (kích thước vector, distance metric). Trong một collection, bạn lưu points — mỗi point gồm id, vector và payload (metadata tùy ý).

Distance metric: phổ biến nhất là Cosine cho semantic search, Euclid cho embedding chuẩn hóa, và Dot cho một số model đặc biệt.

Tạo collection và indexing

from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams

client = QdrantClient("http://localhost:6333")

COLLECTION = "articles"
EMBED_DIM = 1536  # text-embedding-3-small

client.create_collection(
    collection_name=COLLECTION,
    vectors_config=VectorParams(size=EMBED_DIM, distance=Distance.COSINE),
)

Qdrant tự động build HNSW index khi bạn upsert dữ liệu. Bạn cũng có thể cấu hình index parameters như m (số kết nối mỗi node) và ef_construct (độ chính xác khi build) để cân bằng giữa recall và build time.

Upsert dữ liệu và search

from qdrant_client.models import PointStruct
import uuid

def embed(text: str) -> list[float]:
    # Giả lập embedding function
    return [0.0] * EMBED_DIM

points = [
    PointStruct(
        id=str(uuid.uuid4()),
        vector=embed("Qdrant là vector database mã nguồn mở"),
        payload={
            "title": "Giới thiệu Qdrant",
            "category": "database",
            "published": "2026-01-15"
        }
    ),
    PointStruct(
        id=str(uuid.uuid4()),
        vector=embed("RAG kết hợp LLM với retrieval"),
        payload={
            "title": "RAG thực chiến",
            "category": "ai",
            "published": "2026-03-05"
        }
    ),
]

client.upsert(collection_name=COLLECTION, points=points)

# Search
results = client.search(
    collection_name=COLLECTION,
    query_vector=embed("vector database open source"),
    limit=3,
)

for r in results:
    print(r.payload["title"], r.score)

Payload filtering mạnh mẽ

Đây là tính năng mình dùng nhiều nhất. Bạn có thể kết hợp vector search với điều kiện metadata để loại bỏ kết quả không mong muốn trước hoặc sau khi search. 

from qdrant_client.models import Filter, FieldCondition, MatchValue

results = client.search(
    collection_name=COLLECTION,
    query_vector=embed("RAG pipeline"),
    query_filter=Filter(
        must=[
            FieldCondition(
                key="category",
                match=MatchValue(value="ai")
            )
        ]
    ),
    limit=3,
)

Qdrant hỗ trợ nhiều loại payload index: keyword, integer, float, geo, text. Hãy tạo index cho các field thường dùng để filter để tránh full-scan.

Hybrid search: kết hợp semantic + keyword

Dense embedding rất tốt cho semantic similarity, nhưng đôi khi thua từ khóa chính xác (ví dụ: tên sản phẩm, mã lỗi). Qdrant hỗ trợ sparse vectors để kết hợp cả hai: 

from qdrant_client.models import SparseVectorParams, SparseIndexParams

# Tạo collection với cả dense và sparse vectors
client.create_collection(
    collection_name=COLLECTION,
    vectors_config=VectorParams(size=EMBED_DIM, distance=Distance.COSINE),
    sparse_vectors_config={
        "text-sparse": SparseVectorParams(index=SparseIndexParams())
    }
)

Sau đó bạn có thể gọi query_points với prefetch để kết hợp kết quả từ nhiều nguồn vector khác nhau. Đây là cách hiện đại nhất để cải thiện chất lượng RAG mà không cần thay đổi embedding model.

Tips cho production

  • Batch upsert: đừng upsert từng point một. Qdrant xử lý batch vài nghìn points rất tốt.
  • Snapshot: định kỳ tạo snapshot để backup. Qdrant hỗ trợ snapshot toàn bộ collection hoặc một phần.
  • Replication: trong production cluster, bật replication để tăng availability và throughput read.
  • Quantization: dùng scalar hoặc binary quantization để giảm memory footprint khi có hàng trăm triệu vectors.

Kết luận

Qdrant là một lựa chọn vector database đáng cân nhắc cho cả dự án nhỏ và hệ thống production. Với API rõ ràng, hiệu năng tốt và khả năng self-host, nó giúp bạn xây dựng RAG, search và recommendation mà không bị ràng buộc vào dịch vụ độc quyền. Nếu đang làm việc với embeddings, hãy dành một buổi chiều thử Qdrant — có thể bạn sẽ không quay lại giải pháp cũ.

Bạn đang dùng vector database nào? Mình muốn so sánh trải nghiệm Qdrant với Pinecone hay Weaviate từ góc nhìn thực tế.
